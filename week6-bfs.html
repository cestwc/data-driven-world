The Intuition

A graph models relationships.

Analogy:
  • Node: A Person, City, or Webpage.
  • Edge: A Friendship, Road, or Hyperlink.

Unlike charts, graphs don't plot X vs Y. They plot structure.

Formal Definition

A graph $G$ is a pair $(V, E)$ where:

  • $V$ is a set of Vertices (Nodes).
  • $E$ is a set of Edges (Pairs of nodes).

Undirected

Edge $\{A, B\}$
(Mutual, e.g., Siblings)

Directed

Edge $(A, B)$
(One-way, e.g., Follows)

The Problem: Traversal

We are at Node S. We want to:

  1. Find if a path to E exists.
  2. Find the Shortest Path.
  3. Visit every node exactly once.

Mechanism: The Queue

Breadth-First Search (BFS) explores in layers, like a ripple in a pond.

Data Structure: Queue (FIFO)

"First In, First Out" (Like a cafeteria line)

IN ->
C
B
A
-> OUT

BFS Trace: Step 1

  • Start at Node 0.
  • Mark 0 as Visited.
  • Add 0 to Queue.

Queue: [ 0 ]

BFS Trace: Step 2

  • Dequeue 0.
  • Identify Neighbors: 1, 2.
  • Mark visited & Enqueue them.

Queue: [ 1, 2 ]

BFS Trace: Step 3

  • Dequeue 1.
  • Neighbors of 1: 3, 4.
  • Queue now has 2 (from before), then 3, 4.

Queue: [ 2, 3, 4 ]

Algorithm (Pseudocode)


BFS(Graph G, Node start):
    create Queue Q
    mark start as visited
    Q.enqueue(start)

    while Q is not empty:
        current = Q.dequeue()
        
        for neighbor in G.adj[current]:
            if neighbor is not visited:
                mark neighbor as visited
                Q.enqueue(neighbor)
Key takeaway: The Queue ensures we visit all nodes at distance $K$ before distance $K+1$.

Why BFS?

Key Property

BFS guarantees finding the Shortest Path in unweighted graphs.

Real World Uses

  • GPS Navigation: Fewest turns/stops.
  • Social Media: "Degrees of Separation" (LinkedIn connections).
  • Web Crawlers: Exploring links from a home page.

Summary

  • Graphs = Nodes + Edges (Relationships).
  • Adjacency Lists are usually efficient.
  • BFS uses a Queue to explore layer-by-layer.
